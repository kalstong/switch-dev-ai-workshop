What is an LLM?

A Large Language Model is a neural network trained on vast text data. It learns patterns of language and generates text by predicting the most likely next token. It doesn't "know" things — it predicts plausible continuations.

Scale
Billions of parameters trained on trillions of tokens of text
Mechanism
Next-token prediction with emergent reasoning capabilities
Models
Claude, GPT, Gemini, Llama, Mistral and many more

Tokens & Context Windows

Tokens are the atomic units LLMs process — not words, but subword pieces. The context window is how many tokens the model can see at once: its working memory.

Techniques

Prompt Engineering

The skill of writing effective instructions for LLMs. Not magic incantations — clear communication. The same skills that make good specs make good prompts.

  • Zero-shot — Ask the question directly, no examples needed
  • Few-shot — Provide input → output examples before your question
  • Chain-of-thought — Ask the model to reason step by step
  • System prompts — Set the model's role, tone, and constraints upfront

What is an Agent?

An agent is an LLM that can take actions — use tools, make decisions, and work through multi-step tasks autonomously. You give it a goal; it figures out the steps.

Think
→
Act

A chatbot answers a question. An agent completes a task. That's the shift — from conversation to delegation.

RAG — Retrieval-Augmented Generation

LLMs have knowledge cutoffs and hallucinate. RAG grounds them by retrieving relevant data at query time and injecting it into the prompt.

Limitations

Hallucinations & Guardrails

LLMs optimize for plausibility, not truth. They confidently generate text that sounds correct but isn't. This is the central challenge.

Ground It
Use RAG to anchor responses in real data
Verify It
Ask for citations, then check them yourself
Structure It
Use typed outputs and validation schemas
Review It
Human-in-the-loop for critical decisions

Responsibility

Ethics & Responsible AI

You're shaping how this technology is used. Build thoughtfully.

  • Bias — Models inherit training data biases. Audit outputs in sensitive contexts
  • Privacy — Understand data retention before sending sensitive information to APIs
  • Transparency — Users should know when they're interacting with AI
  • Impact — AI changes roles, not eliminates them. Adaptability is the skill
