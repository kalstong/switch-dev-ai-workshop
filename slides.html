Foundations

What is an LLM?

A neural network trained on massive text data to predict the next token. Scale turns pattern matching into something that looks like reasoning.

Trained on text

Books, code, websites — trillions of tokens of human-written text

Billions of parameters

Learned weights that encode patterns in language, logic, and knowledge

Emergent behavior

At sufficient scale, statistical prediction starts to produce reasoning, creativity, and code

Think of it as: autocomplete on your phone, scaled up by a factor of a million — so much scale that it starts to look like thinking.

Foundations

How LLMs Think

Tokens

LLMs read subword units, not words. "understanding" = 2-3 tokens. Context windows and pricing are measured in tokens.

Context Window

How much text the model can "see" at once. Claude: up to 200K tokens (~500 pages). Relevance matters more than volume.

Temperature

Controls randomness. 0 = deterministic, same input → same output. 1 = creative. Low for code, high for brainstorming.

Practical Skills

Prompting Techniques

The #1 skill in working with AI. Not magic — clear communication.

Zero-shot

Just ask directly. Works for simple, well-defined tasks. "Translate this to Spanish."

Few-shot

Give examples first. Dramatically improves quality. "Here are 3 good commit messages: [...] Now write one."

Chain-of-thought

Ask the model to reason step by step. Unlocks complex problem solving. "Think through this step by step."

System prompts

Set persona and constraints upfront. Controls tone, focus, and behavior. "You are a senior security reviewer."

Agents

What is an Agent?

The shift from "AI as a tool" to "AI as a collaborator."

Traditional LLM

  • Stateless: prompt in, response out
  • Can only generate text
  • No memory between calls
  • You do all the work around it

Agent

  • Observe → Think → Act → loop
  • Uses tools: files, commands, APIs
  • Makes decisions autonomously
  • Works through multi-step tasks

Example: "Fix this bug" — a chatbot gives you advice. An agent reads the code, writes a fix, runs the tests, and iterates until they pass.

Why sub-agents?

Single agents hit limits — context overflows, focus drifts. Sub-agents keep each task scoped and efficient.

Fan-out pattern

Run multiple agents in parallel for independent tasks. Each can have different models, tools, and permissions.

Extensibility

Skills & Commands

Turn a general-purpose AI into a domain expert for your codebase.

Skills

Packaged capabilities — like plugins. A skill bundles a prompt, instructions, and tool access for a specific task.

Commands

User-facing entry points that trigger skills. Type a slash command and the agent knows exactly what to do.

// skills are composable /deploy |-- runs /test // ensure tests pass |-- runs /build // compile & bundle |-- runs /security // check for vulnerabilities '-- pushes to production

Infrastructure

MCP — Model Context Protocol

USB for AI. One standard protocol that lets any AI connect to any tool.

Before MCP

  • Every AI tool builds custom integrations
  • N clients × M services = N×M connectors
  • Fragmented ecosystem
  • Vendor lock-in

With MCP

  • Tools expose a standard interface
  • N + M implementations total
  • Any AI client ↔ any MCP server
  • Open protocol, built by Anthropic

Tool Use & Function Calling

The mechanism that lets LLMs interact with the real world. The model is the brain; tools are the hands.

// define a tool { "name": "get_weather", "description": "Get current weather for a city", "parameters": { "city": { "type": "string" } } } // the model decides WHEN to call and WITH WHAT arguments // your code executes — the model never runs code directly

Security: Always validate tool calls. The model suggests actions; your code decides what's actually allowed to execute.

Developer Tools

AI-Assisted Coding

From autocomplete to autonomous agents — the spectrum of AI coding tools.

Autocomplete

Inline suggestions as you type. Fast, low-friction. Copilot, Cody, Supermaven.

Chat

Ask questions, get code snippets. Good for learning and exploration. Claude.ai, ChatGPT.

Agents

Autonomous coding — reads files, writes code, runs tests, iterates. Claude Code, Cursor, Windsurf.

These tools don't replace developers — they amplify you. The developers who learn to work with AI will be the most productive.

Reality Check

Limitations & Guardrails

Trust but verify. AI is a power tool, not an oracle.

Hallucinations

Models confidently generate false information. They don't "know" they're wrong. Always verify critical outputs.

Bias

Training data contains biases. Be especially aware in hiring, medical, and legal contexts.

Context limits

Even 200K tokens isn't infinite. Models can lose track in very long conversations.

Guardrails matter

In production: validate outputs, structured responses, human-in-the-loop for high-stakes decisions.

The rule: You wouldn't ship code without tests — don't ship AI outputs without validation.

Your Future

AI & Your Career

AI won't replace developers — but developers who use AI will outperform those who don't.
  • Invest now: prompt engineering, AI architectures, building with AI APIs, evaluating outputs critically
  • Abstractions are moving up: from writing every line → describing intent and guiding AI
  • New skill: being a great AI collaborator rewards clear thinking, good communication, and engineering judgment
  • Start today: build something with an AI API this week. The learning curve is gentler than you think.
